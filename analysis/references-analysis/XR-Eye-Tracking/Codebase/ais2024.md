## 1. 개요 (Challenge / Dataset / Task / Metric)

### 1.1 챌린지
- **Event-based Eye Tracking Challenge 2024** (CVPR 2024 Workshop "AI for Streaming") 의 3개 서브프로젝트(공식 베이스라인 + 두 팀 제출본) 모음.
  - 근거: `challenge_demo_code/README.md` L2, L16 ("@ CVPR 2024 Workshop AI for Streaming"), L22-28 (인용: *Event-based eye tracking. AIS 2024 challenge survey*, CVPR 2024).
- 응용 동기: 이벤트 카메라는 안구의 빠른 움직임(>300°/s)을 고속·저전력으로 추적 가능 → AR/VR 헤드셋 경량화·저전력화에 적합.
  - 근거: `challenge_demo_code/README.md` L52, L60-64; `ERVT/README.md` L7-13.

### 1.2 데이터셋 (3ET+)
- **3ET+ dataset**, Kaggle 경진대회 `event-based-eye-tracking-ais2024`에서 배포.
  - 근거: `challenge_demo_code/README.md` L17; `eye_track_spatiotemporal/README.md` L7, L18-19.
- 구성: 13명 피험자, 각 2~6 세션. 활동 5종(random, saccades, read text, smooth pursuit, blinks). 압축 .h5 형태로 총 약 1GB.
  - 근거: `challenge_demo_code/README.md` L79.
- 원본 raw event: `(t, x, y, p)` 튜플. p=+1(밝아짐)/-1(어두워짐).
  - 근거: `challenge_demo_code/README.md` L93.
- 라벨: 100Hz로 제공, 각 라벨 `(x, y, close)` — 동공중심 좌표 + 눈감음 이진값(0=open, 1=close). 테스트 평가는 **20Hz**로만 수행.
  - 근거: `challenge_demo_code/README.md` L151-158.
- 분할: 테스트 12 recording, 나머지가 train split (사용자가 train/val 자유 분할).
  - 근거: `challenge_demo_code/README.md` L160-161.

### 1.3 Task / 입력 / 출력
- **입력**: 이벤트 스트림(또는 그로부터 만든 event voxel grid / event frame). 공간 다운샘플 ×8 (640×480 → 80×60) 적용이 베이스라인 기본값.
  - 근거: `challenge_demo_code/README.md` L101-104; `challenge_demo_code/train.py` L144-146 (`transforms.Downsample(spatial_factor=factor)`).
- **출력**: 동공 중심 좌표 `(x, y)`. 제출은 `submission.csv`로 컬럼 `x∈[0,80]`, `y∈[0,60]`.
  - 근거: `challenge_demo_code/README.md` L209; `challenge_demo_code/test.py` L77 (`['row_id', 'x', 'y']`).

### 1.4 메트릭 (p-accuracy / p-error)
- **p10 accuracy** (Kaggle 리더보드): 다운샘플된 80×60 공간에서 예측이 GT로부터 10픽셀 이내인 비율.
  - 근거: `challenge_demo_code/README.md` L211-215; `challenge_demo_code/utils/metrics.py` `p_acc` L6-27 (`torch.norm(dis) < p_tolerance`, `pixel_tolerances=[1,3,5,10]`).
- **averaged euclidean distance** (= p-error, 리포트용): 예측-GT 평균 유클리드 거리.
  - 근거: `challenge_demo_code/utils/metrics.py` `px_euclidean_dist` L63-82.
- 감은 눈(blink) 제외 변형: `p_acc_wo_closed_eye` (눈감음 프레임은 dist=inf 처리).
  - 근거: `challenge_demo_code/utils/metrics.py` L30-60.
- **inference latency**: 리더보드 점수엔 미반영하나 최종 리포트에서 매우 중요(실시간 적용성 평가).
  - 근거: `challenge_demo_code/README.md` L217-218.

## 3. 서브프로젝트별 정밀 분석

각 항목: backbone · neck · head · loss · dataset · train/infer · 이벤트 표현 (라인 근거 포함).

### 3.1 `challenge_demo_code` — 공식 베이스라인 (CNN_GRU)

핵심 소스: `model/BaselineEyeTrackingModel.py`, `train.py`, `test.py`, `utils/metrics.py`, `utils/training_utils.py`, `dataset/custom_transforms.py`, `configs/sliced_baseline.json`.

- **이벤트 표현**: event **voxel grid**. raw event → 시간창 슬라이싱 → Tonic `to_voxel_grid_numpy`로 `n_time_bins` 채널 voxel 변환, 비제로 위치만 per-channel 정규화.
  - 근거: `dataset/custom_transforms.py` `EventSlicesToVoxelGrid.__call__` L111-134 (`to_voxel_grid_numpy(...)`, L125-132 정규화).
- **backbone**: 3-layer 2D-CNN. `conv1: n_time_bins→32`, `conv2: 32→64`, `conv3: 64→32`, 각 ReLU, 그 뒤 `MaxPool2d(k=3,s=2)`.
  - 근거: `model/BaselineEyeTrackingModel.py` L11-14, L26-32.
- **neck(시간 모델링)**: 단일층 `GRU(input=36192, hidden=128)` (시퀀스 차원으로 펼친 CNN 특징을 입력).
  - 근거: 동 파일 L15, L34-36. (입력 36192 = pool 후 spatial×channel flatten 값 하드코딩.)
- **head**: `Linear(128→2)` → 프레임별 `(x,y)`.
  - 근거: 동 파일 L16, L38-39. 출력 shape `(B, seq_len, 2)`.
- **loss**: `mse` 또는 `weighted_mse`. weighted_mse는 가로/세로 비율 `(W/H, 1)` 가중치로 좌표 손실 균형.
  - 근거: `train.py` L125-131; `utils/metrics.py` `weighted_MSELoss` L85-99. 기본 config는 `weighted_mse` (`configs/sliced_baseline.json` L24).
- **dataset 파이프라인**: Tonic `ThreeETplus_Eyetracking` → spatial Downsample(factor) → label transform(`ScaleLabel`+`TemporalSubsample`+`NormalizeLabel`) → `SliceByTimeEventsTargets`(시간창 슬라이싱) → `EventSlicesToVoxelGrid` → `SlicedDataset` → `DiskCachedDataset` → `DataLoader`.
  - 근거: `train.py` L137-185. 라벨 정규화 pseudo_width/height=640·480·factor (L140).
  - 슬라이싱 산식: `slicing_time_window = train_length*(10000/temp_subsample_factor)`, stride 동일 산식 (L154-155). README L108-114 동일 설명.
- **train/infer**: `train.py` MLflow 로깅, Adam, val_loss 최저 모델 저장 + top-k 체크포인트 유지(`top_k_checkpoints`). `test.py`는 `include_incomplete=True`로 테스트 슬라이싱, 출력 좌표를 80×60 공간으로 환산(`output * (W*factor, H*factor)`)해 csv 기록.
  - 근거: `train.py` L32-90, L122-123; `utils/training_utils.py` L85-99; `test.py` L47-94 (L86 좌표 환산).
- **성능(자기보고)**: CNN_GRU, Averaged Distance 8.99, P10 0.63 (RTX 3090).
  - 근거: `challenge_demo_code/README.md` L173.

### 3.2 `ERVT` — Efficient Recurrent Vision Transformer

핵심 소스: `model/RVT.py`, `train.py`, `test.py`, `bench.py`, `dataset/ThreeET_plus.py`(미열람 — Tonic 데이터셋 래퍼, 베이스라인과 동일 계열로 추정), `dataset/custom_transforms.py`, `utils/metrics.py`, `utils/training_utils.py`, `configs/rvt.json`.

설계 출발점: Gehrig & Scaramuzza의 **RVT (arXiv:2212.05598)** 를 단순화. 4-stage → 2-stage, Grid+Block attention → 단일 Full Self-Attention.
  - 근거: `ERVT/README.md` L15-16; `model/RVT.py` L1-4 헤더 주석.

- **이벤트 표현**: voxel grid (베이스라인과 동일 Tonic 파이프라인). `n_time_bins=3`, `in_channels=3`, per-channel normalize=true.
  - 근거: `configs/rvt.json` L48-50; `dataset/custom_transforms.py` `EventSlicesToVoxelGrid` L98-134. (대체 표현 `EventSlicesToSpikeTensor`도 구현되어 있음 — L136-226, SNN/스파이크용, 기본 미사용으로 추정.)
- **backbone**: `RVT` = 시간 루프 안에서 `RVTBlock` 스택 반복. 각 `RVTBlock` 구성:
  1. **entry conv** `Conv2d(in→out, k, stride, pad=k//2)` + LayerNorm,
  2. **Full Self-Attention**(잔차) + LayerNorm + **MLP(GLU 게이트형)**,
  3. **DWSConvLSTM2d**(깊이분리 ConvLSTM)로 시간 상태 갱신.
  - 근거: `model/RVT.py` `RVTBlock.__init__` L21-35, `forward` L37-53; `RVT.forward` 시간 루프 L73-106 (L84 `for t in range(N)`, L90-95 stage별 lstm_state 전달).
  - 2 stage 설정: stage0 `k=7,out=32,stride=2,dim_head=32`, stage1 `k=3,out=64,stride=2,dim_head=32`.
    - 근거: `configs/rvt.json` L14-27.
- **Self-Attention**: `SelfAttentionCl`(channels-last MHSA, `num_heads=dim//dim_head`, scale=dim_head^-0.5, dropout 0.5) + `LayerScale`(γ 초기 1e-5)로 감싼 `FullSelfAttention`.
  - 근거: `model/RVT.py` `SelfAttentionCl` L191-219, `FullSelfAttention` L221-236, `LayerScale` L243-253.
- **ConvLSTM(neck/recurrence)**: `DWSConvLSTM2d` — `[x;h]` concat 후 7×7 depthwise conv + 1×1 conv로 4개 게이트 생성, dropout 0.2.
  - 근거: `model/RVT.py` L130-184 (L144-148 dws conv, L174-182 게이트/셀 갱신).
- **head**: `LinearHead` — 마지막 stage 출력을 flatten 후 `Linear(dim→2)`. dim은 stride 누적과 공간크기로 계산.
  - 근거: `model/RVT.py` `LinearHead` L108-125 (L118-119 dim 산식), `RVT.forward` L99-105.
- **loss**: `weighted_rmse` 기본(가중 MSE의 sqrt). `weighted_mse`/`mse`도 선택 가능.
  - 근거: `configs/rvt.json` L53; `train.py` L99-108; `utils/metrics.py` `weighted_RMSE` L103-117.
  - 주의(버그): `weighted_RMSE` reduction='sum' 분기에 `torch.sqrt*(...)` 오타(연산자 누락). mean 분기만 정상 동작.
    - 근거: `utils/metrics.py` L115.
- **dataset/augmentation**: 베이스라인 동일 슬라이싱 + `MemoryCachedDataset` + `RandomSpatialAugmentor`(h-flip, gaussian noise, time-reversal, random-time-shift). 기본 config는 hflip 0.5, noise 0.5만 활성, time-reversal/shift=0.
  - 근거: `train.py` L153-155; `dataset/custom_transforms.py` `RandomSpatialAugmentor` L372-442; `configs/rvt.json` L54-61.
- **train/infer**: Adam + `ExponentialLR(gamma=0.98)`, 150 epoch, **batch_size=1**, `torchinfo.summary`로 파라미터 출력. config에 `tbptt:15`(truncated BPTT 길이 추정 — 코드 내 직접 사용처 확인 불가). `test.py`는 `model(data)` → `(coords, lstm_states)` 중 coords를 80×60으로 환산해 csv 기록. `bench.py`는 `torch.compile` 후 1000회 `CudaTimer`로 latency 측정.
  - 근거: `train.py` L92-97, L31-56; `configs/rvt.json` L13, L28-31; `test.py` L82-94; `bench.py` L34-43.
- **성능(자기보고)**: 2-layer Efficient RVT — P10 97.6%, params 149,570, mult-adds 36.96M, 1.06ms (RTX 3060 Laptop, bs=1).
  - 근거: `ERVT/README.md` L21, L24-25.

### 3.3 `eye_track_spatiotemporal` — TENNs-Eye (Brainchip)

핵심 소스: `tenn_model.py`, `losses.py`, `eye_dataset.py`, `train.py`, `generate_submission.py`, `generate_val_results.py`, `config.yaml`, `config_submission.yaml`.

성격: Brainchip **TENNs**(Temporal Neural Networks) 계열의 경량 **인과적(causal) 시공간 conv 네트워크**. FIFO 버퍼로 실시간 스트리밍 추론 지원. AIS2024에서 p10 3위, 그 외 메트릭 1위 자기보고.
  - 근거: `eye_track_spatiotemporal/README.md` L1-7.

- **이벤트 표현**: raw event를 직접 frame으로 변환. `events_to_frames`가 bilinear/nearest/**causal_linear** 보간으로 공간·시간 다운샘플을 한 번에 수행 → `(T, 2, H, W)` (극성 2채널). 핵심 차별점: `causal_linear` 모드는 시간 보간이 인과적(미래 비참조).
  - 근거: `eye_dataset.py` `events_to_frames` L37-94 (L67-80 causal_linear, L77 `td = td1.repeat(4)  # causal`). config 기본 frames spatial_downsample (5,5), time_window 10000us.
  - 근거: `config.yaml` L1-9 (events_interpolation: causal_linear), `config_submission.yaml` L5 (bilinear).
- **backbone**: `TennSt` — 10개 블록 시퀀스, `[True,False]*5`로 **TemporalBlock**(시간 conv)과 **SpatialBlock**(공간 다운 conv)을 교대. 채널 `[2,8,16,32,48,64,80,96,112,128,256]`. 마지막 4개 블록 depthwise(`n_depthwise_layers=4`).
  - 근거: `tenn_model.py` `TennSt.__init__` L153-198 (L167-168 depthwise/temporal 패턴, L170-184 backbone 구성); `config.yaml` L12-14.
- **SpatialBlock**: `Conv3d` 커널 `(k,3,3)` stride `(1,2,2)` → 공간 ½ 다운샘플. depthwise면 groups=in_channels + pointwise conv. norm은 `mixed`(BN/GN) 선택.
  - 근거: `tenn_model.py` L30-86.
- **TemporalBlock**: `Conv3d` 커널 `(k,1,1)`(또는 full_conv3d면 `(k,3,3)`), 입력을 **왼쪽 시간 패딩**(`F.pad(..., (kernel-1,0))`)으로 인과 처리. 스트리밍 모드에서는 FIFO 버퍼로 1프레임씩 처리.
  - 근거: `tenn_model.py` L89-150 (L143 causal pad, L146-150 `_streaming_forward` FIFO).
- **CausalGroupNorm**: 시간 통계 미사용 GroupNorm(인과성 유지). `(B,T,C,H,W)`를 `(B*T,C,H,W)`로 펴서 정규화.
  - 근거: `tenn_model.py` L10-21.
- **head (2종)**:
  - `detector_head=True` (CenterNet식): TemporalBlock + Conv3d(3×3) + ReLU + Conv3d(→3). 3채널 = pupil heatmap + (x_mod, y_mod). argmax로 픽셀 위치, sub-pixel 오프셋 보정.
    - 근거: `tenn_model.py` L186-192; `losses.py` `process_detector_prediction` L137-165 (L152 argmax, L162-163 sub-pixel).
  - `detector_head=False` (회귀): 공간 평균 후 `Conv1d → ReLU → Conv1d(→2)`.
    - 근거: `tenn_model.py` L194-198, L212-216 (L216 `.mean((-2,-1))`).
  - 기본 config는 `detector_head: true`.
    - 근거: `config.yaml` L15, `config_submission.yaml` L15.
- **loss**: detector_head면 `tracking_loss`(focal loss + smooth_l1 sub-pixel), 아니면 `regression_loss`(smooth_l1). 추가로 ReLU 활성에 L1 정규화(`RegularizationLoss`, 활성 희소화 유도). blink/범위밖 좌표는 `valid_mask`로 제외.
  - 근거: `losses.py` `regression_loss` L79-87, `tracking_loss` L90-120 (L114-118 focal), `RegularizationLoss` L56-76 (ReLU hook L63-65), `Losses` L123-134.
- **dataset/augmentation**: `EyeTrackingDataset`이 h5 raw event + label.txt 로드, segment 단위 분할. 증강: `EventRandomAffine`(회전±15°, translate, scale 0.8~1.2, spatial jitter), temporal flip(+극성 flip), temporal scale(0.8~1.2), temporal shift. val_files 하드코딩(9개 recording).
  - 근거: `eye_dataset.py` `EventRandomAffine` L97-153, `EyeTrackingDataset` L156-313 (L14 val_files, L233-235 affine, L252-257 temporal flip, L270-286 shift/scale, L297-310 label 보간).
- **train/infer**:
  - 학습: PyTorch **Lightning** + **Hydra**. `torch.compile(TennSt)`, AdamW(lr=2e-3, wd=1e-3) + `LinearWarmupCosineAnnealingLR`, grad clip=1, 16-mixed precision, val_metric(p10) 최대 체크포인트. 학습 종료 후 `check_val_score`로 p10/거리/MACs/event density를 csv 기록.
    - 근거: `train.py` `CustomModule` L35-128, `configure_optimizers` L115-122, `main` L131-185.
  - 추론(제출): `generate_submission.py`가 **GPU 끄고 streaming 모드**로 1프레임씩 추론(FIFO). 예측을 `::5`로 20Hz 다운샘플, 80×60 환산 후 csv.
    - 근거: `generate_submission.py` L14-17(`CUDA_VISIBLE_DEVICES=''`), L20-32 `streaming_inference`, L55-70.

## 4. 알고리즘·데이터 표현 비교

### 4.1 이벤트 표현
| 서브 | 표현 | 변환 위치 | 인과성 |
|---|---|---|---|
| baseline | voxel grid (Tonic) | `EventSlicesToVoxelGrid` (custom_transforms L98-134) | 시간창 내 voxel, 비인과 |
| ERVT | voxel grid (Tonic, n_bins=3) | 동일 transform | 비인과 (voxel) |
| TENNs-Eye | event frame(극성 2ch), bilinear/nearest/**causal_linear** | `events_to_frames` (eye_dataset L37-94) | **causal_linear 인과 지원** |

- 공통: 입력 640×480 → 다운샘플. baseline/ERVT는 spatial_factor=0.125(×8→80×60), TENNs-Eye는 spatial_downsample=(5,5)(→128×96).
  - 근거: `configs/rvt.json` L32; `config.yaml` L4.
- TENNs-Eye는 추가로 `EventSlicesToSpikeTensor`(spike tensor) 옵션도 ERVT custom_transforms에 존재(L136-226) — SNN 호환 표현이나 기본 미사용 추정.

### 4.2 시간 모델링 메커니즘
| 서브 | 시간 모델링 | 비고 |
|---|---|---|
| baseline | **GRU** (1층) | CNN 특징을 시퀀스로 입력 |
| ERVT | **ConvLSTM**(DWSConvLSTM2d) + **Self-Attention** + entry Conv | RVT 단순화(2-stage), 명시적 시간 루프(L84) |
| TENNs-Eye | **인과적 Temporal Conv3d** + FIFO 버퍼 | recurrence 없이 causal conv로 시간 처리, 스트리밍 |

- **SSM**: 세 서브 모두 명시적 State Space Model(S4/Mamba류) 미사용 — 확인 불가/해당 없음.
- **RVT 계열**: ERVT만 RVT 직계(Conv+Attn+ConvLSTM 블록). baseline은 순수 CNN+RNN, TENNs-Eye는 순수 시공간 CNN.
- 패러다임 차이: ERVT는 명시적 recurrent state(`lstm_states`)를 시간축 루프로 전달(stateful), TENNs-Eye는 학습 시 3D conv 전체 시퀀스 처리 + 추론 시 FIFO로 등가 스트리밍(stateless-by-design causal conv).
  - 근거: `model/RVT.py` L73-106 vs `tenn_model.py` L139-150, `generate_submission.py` L20-32.

### 4.3 출력 헤드 패러다임
- baseline/ERVT: 직접 좌표 **회귀**(Linear→2).
- TENNs-Eye: **detector(CenterNet식 heatmap+offset)** 기본, 회귀 헤드는 옵션. heatmap argmax+sub-pixel이 좌표 정밀도에 유리할 수 있음(추정).
  - 근거: `losses.py` L137-165.

## 7. 강점·한계

### 7.1 `challenge_demo_code` (baseline)
- 강점: 표준 Tonic 파이프라인·캐싱·MLflow 로깅 완비된 재현 가능한 스타터. 단순 CNN+GRU로 진입장벽 낮음.
- 한계: P10 0.63로 정확도 낮음. GRU 입력 36192 하드코딩(해상도/채널 변경 시 깨짐 — `BaselineEyeTrackingModel.py` L15). voxel grid는 비인과·고연산.

### 7.2 `ERVT`
- 강점: 150k params로 97.6% p10 — 파라미터 효율 최상위급. 1.06ms 저지연(자기보고). LayerScale/GLU 등 안정화 기법 적용.
- 한계: ConvLSTM+Attention의 명시적 시간 루프(batch_size=1)로 학습 병렬성 제약. `weighted_RMSE` sum 분기 버그(L115). voxel grid 입력은 여전히 비인과적 표현. dropout 0.5/0.6로 강한 정규화 의존.
  - 근거: `configs/rvt.json` L12(drop_prob 0.6), `model/RVT.py` L205.

### 7.3 `eye_track_spatiotemporal` (TENNs-Eye)
- 강점: **인과적 시공간 conv + FIFO 스트리밍**으로 실시간 on-device 추론 친화적(CPU만으로 추론). 활성 희소도/MACs 정량 측정 코드 내장 → HW 효율 분석 가능. CenterNet 헤드 + 정교한 augmentation으로 정확도 1위(자기보고).
  - 근거: `generate_submission.py` L14-17, `tenn_model.py` L131-150, `generate_val_results.py` L127-141.
- 한계: 의존성 무거움(Lightning/Hydra/bolts). `torch.compile` 의존. detector 헤드의 heatmap 후처리(argmax) 추가 연산. 일부 코드 주석처리/실험 잔재(예: train.py `on_train_end` 주석 L68-91).

## 8. 우리 프로젝트 시사점 — "XR 시선추적 + FPGA 저지연 on-device 가속" (추정)

> 우리 프로젝트 목표는 "추정"임(PRJXR-HBTXR 폴더명·동반 가속기 코드베이스로부터 추론). 아래는 경량화·양자화·FPGA 이식 관점 시사점.

1. **TENNs-Eye가 FPGA 이식 1순위 후보(추정)**: 인과적 conv + FIFO 버퍼 구조는 스트리밍 데이터패스/라인버퍼 기반 FPGA 매핑과 자연스럽게 대응. `_streaming_forward`의 FIFO(L82-86, L146-150)는 HW 시프트레지스터/순환버퍼로 직역 가능. recurrence(LSTM state) 없이 고정 커널 conv만 사용 → 파이프라인화·정적 스케줄링 용이.
2. **희소도/MACs 계측 코드 재활용**: `MacsEstimationHook`(losses.py L15-53)과 event-density 측정(generate_val_results.py L75-81, L127-141)은 FPGA 리소스/전력 예측 및 sparsity-aware 가속(zero-skipping) 설계 입력으로 직접 활용 가능.
3. **양자화 대상**: ERVT(150k)·TENNs-Eye 모두 소형 → PTQ/QAT INT8 후보. 단 ERVT의 Self-Attention(softmax)·LayerNorm은 FPGA에서 비용이 큼 → TENNs-Eye의 conv+GroupNorm/BN 조합이 정수화·접기(folding)에 유리(추정).
4. **입력 표현 선택**: voxel grid(baseline/ERVT)는 부동소수 정규화(per-channel) 포함 → on-device 전처리 부담. TENNs-Eye의 event-frame(누적/극성 2채널)이 정수 누적기로 HW 구현 단순(추정). `causal_linear`는 미래 비참조라 실시간 파이프라인에 적합.
5. **저지연 목표 정합성**: 챌린지 자체가 latency를 핵심 리포트 지표로 명시(README L217-218)하므로, 우리 FPGA 가속 목표(저지연 on-device)와 평가 축이 일치. ERVT 1.06ms(GPU)·TENNs-Eye CPU 추론 가능은 FPGA 목표 설정의 baseline 수치로 사용 가능.
6. **정확도-비용 트레이드오프 데이터 확보**: 세 모델이 동일 데이터/메트릭(p10, euclidean) 위에 있어 가속 후 정확도 손실 평가의 공정한 기준선 제공.
