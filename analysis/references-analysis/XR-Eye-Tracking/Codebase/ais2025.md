## 1. 개요 (챌린지 / 데이터셋 / 입력·출력 / 메트릭)

- **챌린지**: Event-based Eye Tracking Challenge 2025 (CVPR 2025 Event-based Vision Workshop, AIS/3ET+ 계열). 주최: Leiden Univ., TU Delft, DVSense, Prophesee 등 (확인: `3et_challenge_2025/README.md:1-15`). 기간 2025-02-15 ~ 2025-03-15, 1위 상품 Meta Quest 3 (확인: README:20-28).
- **데이터셋 (3ET+ 2025)**: 13명 피험자, 각 2-6 세션, 5종 활동(random / saccades / read text / smooth pursuit / blinks). 압축 .h5로 약 1GB. Kaggle "event-based-eye-tracking-cvpr-2025"에서 배포 (확인: README:122-123, 184). 원 센서 해상도 640x480, 폴라리티 2채널 (확인: `dataset/ThreeET_plus.py:36` `sensor_size = (640, 480, 2)`).
- **입력**: 이벤트 스트림. 각 이벤트 = (t[us], x, y, p). 폴라리티 p는 원본 0/1 → 코드에서 -1/+1로 변환 (확인: `ThreeET_plus.py:37, 85-86`). 모델 입력은 raw event를 시간윈도우로 슬라이스 후 **voxel grid / binary(누적 프레임) 표현**으로 변환한 텐서 (확인: `dataset/custom_transforms.py:8-65, 156-192`).
- **출력**: 동공 중심 좌표 (x, y). 라벨은 100Hz, (x, y, close) 형태이며 close는 눈감음 binary (확인: README:255-262). 다운샘플 학습 시 공간 좌표는 640x480 → 80x60 (spatial_factor=0.125) (확인: README:206-208, `configs/train_baseline.json:11`).
- **메트릭**:
  - 공식 Kaggle 평가 = **Average Euclidean Distance (p-error)**: `sqrt((x_true-x_pred)^2 + (y_true-y_pred)^2)`의 전체 평균. x∈[0,80], y∈[0,60] (확인: README:174-178).
  - 보조 메트릭 **p-accuracy(pₜ)**: 픽셀 허용오차 tolerance(1/3/5/10/15) 이내 정답 비율 (확인: `utils/metrics.py:6-27`, `p_acc`).
  - 베이스라인 성능: CNN_GRU = 7.91384 Avg Euclidean Distance (RTX 4090 Mobile) (확인: README:154-156).
  - Top-10 모델은 **Neurobench로 메모리 풋프린트 평가**, 최저 메모리 팀에 워크숍 논문 보너스 (확인: README:180) → 경량화가 명시적 부상 목표 (확인).

## 3. 서브프로젝트별 정밀 분석

### 3.1 `3et_challenge_2025` — 2025 공식 베이스라인

- **역할**: 주최측 제공 시작점. 기여자 Zuowen Wang, Chang Gao (확인: `train.py:7-9`).
- **Backbone**: `CNN_GRU` (확인: `model/BaselineEyeTrackingModel.py:4-40`).
  - conv1(n_time_bins→32) → conv2(32→64) → conv3(64→32), 각 3x3, ReLU, 마지막 MaxPool(3,2) (라인 11-32).
  - 시퀀스 평탄화 후 `GRU(input_size=36192, hidden=128, num_layers=1, batch_first)` (라인 15, 34-36).
  - 입력 shape (B, seq, C, H, W) → (B*seq, C, H, W)로 CNN 처리 후 (B, seq, -1)로 GRU (라인 19-36).
- **Neck**: 없음(단일 GRU가 시간 모델링 겸함). (확인)
- **Head**: `fc = Linear(128, 2)` → (B, seq, 2) 좌표 회귀 (라인 16, 38-39). 좌표 정규화 회귀(분류 아님). (확인)
- **Loss**: `weighted_MSELoss` (가로:세로 비율 가중) 또는 plain MSE 선택 (확인: `train.py:137-143`, `utils/metrics.py:85-99`). 가중치 = (sensor_width/sensor_height, 1) = (640/480, 1) (확인: train.py:140).
- **Dataset**: `ThreeETplus_Eyetracking` (Tonic Dataset 상속). .h5에서 events 로드, p를 -1/+1로 변환, label.txt에서 (x,y,close) 파싱 (확인: `dataset/ThreeET_plus.py:8-103`). split별 파일목록 = `train_files.txt/val_files.txt/test_files.txt` (라인 58-74). test는 label_zeros.txt placeholder (라인 73-74).
- **이벤트 표현**: **Voxel Grid** (Zhu et al. 2019 event volume, 시간축 bilinear interpolation). `custom_to_voxel_grid_numpy` (확인: `custom_transforms.py:8-65`). 파이프라인: `SliceByTimeEventsTargets`(raw를 sub-sequence로) → `SliceLongEventsToShort`(10000us/tsf 윈도우) → `EventSlicesToVoxelGrid`(n_time_bins voxel) (확인: `train.py:167-183`, `custom_transforms.py:67-192`).
- **Train/Infer**: `train.py` — Adam, MLflow 로깅, Tonic `SlicedDataset` + `DiskCachedDataset` 캐싱 (확인: train.py:135, 186-197). 설정 `configs/train_baseline.json`: lr=0.001, epochs=20, batch=20, spatial_factor=0.125, n_time_bins=3, train_length=30, loss=weighted_mse (확인). 추론 `test.py` + `test_config.json` (tsf는 반드시 1.0, README:220 확인).
- **명령**: `python train.py --config train_baseline.json` / `python test.py --config test_config.json --checkpoint [PATH]` (확인: README:127-128, 167-168).

### 3.2 `Event-based-Eye-Tracking-Challenge-Solution` — 1위 솔루션

- **역할**: CVPR 2025 챌린지 1위 (확인: README:1 "1st Solution"). 베이스라인 코드 구조를 확장.
- **Backbone**: `Model` in `model/CNN_GRU_base.py:11-73`.
  - **3단 2D CNN(BatchNorm+ReLU+AvgPool)**: conv1(2→32, 7x7, AvgPool3) → conv2(32→128, 5x5, AvgPool3) → conv3(128→512, 5x5, Dropout) (확인: 라인 16-27). 입력 채널 **2 고정**(binary 표현의 2 폴라리티, 라인 16) — voxel n_time_bins이 아닌 binary map 사용과 정합 (test.sh map_type=binary, 확인).
  - `AdaptiveAvgPool2d((4,4))` → `SpatialDropout(0.2)` → flatten 4*4*512=8192 (확인: 라인 28-29, 60-64).
- **Neck (핵심)**: **BiGRU + Transformer 직렬**.
  - `GRU(input_size=8192, hidden=128, num_layers=1, bidirectional)` → 256 차원 (확인: 라인 38, 66).
  - `Transformer(num_heads=4, num_layers=6, attn_size=64, widening=4)` (확인: 라인 40-46, 194-234).
  - **상대위치 attention = ALiBi** (`MultiHeadAttentionRelative`, 양방향 bias forward/backward, `get_alibi_slope`/`get_relative_positions`) (확인: 라인 104-187). forward/backward bias를 head 절반씩 분할해 인과/역인과 마스킹 (라인 178-184) → 시간 양방향 시퀀스 모델링.
  - 주석에 `# self.mamba = ResidualBlock(...)` 및 EfficientNet encoder 잔재 존재 → **Mamba/EfficientNet은 실험 후 비활성, 최종은 Transformer 채택** (확인: 라인 30-39, 69 주석).
- **Head**: `fc = Sequential(Linear(256,64), ReLU, Linear(64,2))` → 좌표 **회귀** (확인: 라인 47-49, 72). (베이스라인과 동일한 회귀 방식, 분류 아님.)
- **Loss**: `weighted_MSELoss` (베이스라인과 동일, 확인: `train.py:71-77`, `utils/metrics.py` 동일 구현).
- **Dataset/이벤트 표현**: `EventSlicesToMap` 으로 **voxel 또는 binary 표현 선택 가능** (확인: `dataset/custom_transforms.py:282-334`). map_type='binary'일 때 Tonic `to_frame_numpy` → `to_bina_rep_numpy(n_bits=n_time_bins)` 사용(이진 누적표현) (라인 310-322). 실제 실행은 **binary** (확인: `test.sh:17`, `train.sh:17` map_type=binary, n_time_bins=4).
- **데이터 증강 `Jitter`** (DiskCachedDataset transforms로 주입, 확인: train.py:117): x/y shift(circular), x/y flip, t shift, random cutout — 라벨 좌표도 함께 변환 (확인: `custom_transforms.py:149-280`). mix_flag 기반 patch-mixup 코드는 존재하나 미사용(`mix_flag=None`, 라인 168, 244).
- **학습 디테일**: Adam, **CosineAnnealingWarmRestarts(T_0=8, T_mult=2)** 스케줄러 (확인: `train.py:29`), `nn.DataParallel` (라인 65). epochs=800, train_length=45, stride=10, spatial_factor=0.125 (확인: `train.sh:6-19`). val 기준 best = p_error_all 최저 시 체크포인트 저장 (라인 37-42).
- **추론/제출**: `test.py` — output을 (640*factor, 480*factor) 곱하고 `* 0.125/factor`로 80x60 좌표계 보정 (확인: test.py:77-78, 166-167), GPU 50회 warm-up 후 inference time 측정 (라인 144-147), `inference_time.txt` 기록 → **저지연 의식** (확인). tsf=0.2일 때 5-fold 보간 TTA 분기 존재 (라인 25-107).
- **명령**: `./train.sh`, `./test.sh` (확인: README:13-36, 두 .sh). 사전학습 가중치 `ckpt/final.pth` 다운로드 후 test (확인: test.sh:9).

### 3.3 `tdtracker` — 3위 솔루션 TDTracker

- **역할**: CVPR 2025 챌린지 3위 "Exploring Temporal Dynamics in Event-based Eye Tracker" (확인: README:1-2). MambaPupil/TENNs(2024)에서 아이디어 차용 명시 (확인: README:43-45).
- **Backbone (핵심: spatial/temporal 분리 3D-CNN)**: `Model` in `models/TDTracker.py:9-77`.
  - 3 stage, 각 stage = **spatial conv (1×k×k, 시간축 보존) + temporal conv (3×3×3, 시간축 혼합)**:
    - conv1_spatial(2→32, 1×7×7) + conv1_temporal(32→32, 3×3×3, AvgPool(1,3,3)) (확인: 라인 14-20)
    - conv2_spatial(32→64, 1×5×5) + conv2_temporal(64→64, 3×3×3, AvgPool(1,3,3)) (라인 22-28)
    - conv3_spatial(64→128, 1×5×5) + conv3_temporal(128→128, 3×3×3, Dropout) (라인 30-36)
  - 입력 채널 2(폴라리티), Conv3d 사용 → (B, C=2, T, H, W) (확인: 라인 14, 58-59 permute).
  - `AdaptiveAvgPool3d((128,4,4))` → `SpatialDropout(0.5)` → flatten 128*4*4=2048 (확인: 라인 42-43, 67-69).
  - **FFT temporal layer** 정의됨(`fftlayer_temporal`, rfft→complex weight→irfft) 그러나 forward에서 주석처리 → **미사용**(확인: 라인 38-39, 70-71, 79-89).
- **Neck (핵심: GRU + Mamba SSM)**:
  - `LayerNorm(2048)` → `GRU(2048→128, bidirectional)` → 256 (확인: 라인 44-45, 72-74).
  - `mamba = ResidualBlock(MambaConfig(d_model=256))` — **순수 PyTorch Mamba 구현** (선택적 selective scan: pscan 병렬 또는 `selective_scan_seq` 순차) (확인: 라인 46, 74, 121-419). step()/ssm_step()로 RNN형 상수시간 추론 캐시까지 구현 (라인 317-381) → **on-device 스트리밍 추론 친화** (확인).
- **Head (핵심: SimDR 좌표 분류)**: 회귀가 아니라 **per-axis 분류**.
  - `fc_1: Linear(256→128)→ReLU→Dropout→Linear(128→80)` (x축 80 bin), `fc_2: ...→Linear(128→60)` (y축 60 bin) (확인: 라인 47-56, 75-76). 출력 (predict_w[80], predict_h[60]) (라인 77).
- **Loss**: `KLDiscretLoss` (SimDR, leeyegy/SimDR 인용) — GT 좌표를 1D Gaussian heatmap(variance=1.0)으로 만들어 LogSoftmax+KLDiv (확인: `metrics.py:104-149`). 디코딩 `decode_batch_sa_simdr`: argmax bin → /80, /60 정규화, avg_pool1d+softmax로 **confidence(prob)** 산출 (확인: `metrics.py:151-166`).
- **Dataset/이벤트 표현**: 오프라인 전처리로 .h5 생성. `dataprocess/3etplus.py`:
  - 10000us(=100Hz) 슬라이딩 윈도우로 슬라이스 → Tonic `ToFrame(sensor_size=(80,60,2), n_time_bins=4)` → `ToBinaRep(n_frames=1, n_bits=4)` = **binary(이진 누적) 표현** (확인: 라인 137-148). 출력 frame shape (T, 2, 60, 80) (라인 170-171, 278-280).
  - **오프라인 증강(학습셋만)**: flip_x / flip_y / rotate(-15~15도) 4배 확장 (확인: 라인 174-200, 235-255).
  - group_size=100(시퀀스 길이), step_size 50(train)/100(test) (확인: 라인 262-269).
  - **눈감음 라벨 정제**: 연속 동일좌표면 close=1로 간주(라인 56-70) → 후처리 근거.
- **Train/Infer**: `train.py` — AdamW(lr=0.002, wd=1e-4) + CosineAnnealingWarmRestarts(T_0=8) (확인: 라인 178-179), TensorDataset(.h5 전체 메모리 로드) (라인 158-166), TensorBoard 로깅, p_error 최저 시 저장 (라인 194-199). `provider_data.py`로 .h5 frames/label 로드 (확인: 라인 22-28).
- **후처리(핵심 성능 기여)**: `post_process.py` — (1) confidence(prob)<0.5 샘플을 직전 신뢰 샘플로 대체, (2) num_events_y<0.09(눈감음 추정) 구간을 binary_dilation으로 확장해 직전 값으로 대체 (확인: 라인 15-34). README: test.py MSE 1.5532 → post_process 후 **1.4932** (확인: README:28-34).
- **명령**: `cd dataprocess; python 3et_plus.py` → `python train.py` → `python test.py` → `python post_process.py` (확인: README:15-34). (README는 `3et_plus.py`로 표기하나 실제 파일명은 `3etplus.py` — 사소한 불일치, 확인 불가하나 명백한 오타로 추정.)

## 4. 알고리즘·데이터표현 비교

### 4.1 이벤트 표현
| 서브 | 표현 | 변환 근거 |
|---|---|---|
| 베이스라인 | **Voxel Grid** (시간축 bilinear, n_time_bins=3) | `custom_transforms.py:8-65`, `train_baseline.json:22` |
| 1위 | **Binary(이진 누적 프레임, n_bits=4)** (voxel도 선택 가능) | `custom_transforms.py:308-322`, `test.sh:17` |
| TDTracker | **Binary(ToFrame+ToBinaRep, n_bits=4)** 오프라인 .h5 | `dataprocess/3etplus.py:137-148` |

- 공통: spatial_factor=0.125로 640x480 → **80x60** 공간 다운샘플 (확인: 세 곳 모두). 시간 윈도우 기본 10000us = 100Hz 라벨 정렬 (확인).
- 베이스라인은 voxel의 연속값, 상위 솔루션 2개는 **binary 표현**으로 수렴 → 노이즈 강건성/경량 입력 선호 (추정).

### 4.2 시간 모델링 백본 (ConvLSTM / SSM / Mamba / Transformer / GRU)
| 서브 | 공간 백본 | 시간 모델링 | 헤드 방식 |
|---|---|---|---|
| 베이스라인 | 2D CNN(3층) | **단일 GRU** | 좌표 회귀 (Linear→2) |
| 1위 | 2D CNN(3층, ch 512) | **BiGRU → Transformer(ALiBi, 6층)** | 좌표 회귀 (256→64→2) |
| TDTracker | **3D CNN(spatial/temporal 분리)** | **BiGRU → Mamba(SSM)** | **SimDR 분류(80+60 bin)** |

- **ConvLSTM**: 원조 3ET 논문(Chen 2023, change-based ConvLSTM)에서 사용되나, ais2025 내 3개 솔루션 코드에는 ConvLSTM 미구현 — GRU/Transformer/Mamba로 대체 (확인: 세 모델 파일에 ConvLSTM 없음).
- **SSM/Mamba**: TDTracker가 실제 사용(확인). 1위는 Mamba를 코드에 두되 **주석 비활성**, 최종 Transformer 채택 (확인: CNN_GRU_base.py:39, 69). → **Transformer(1위) vs Mamba(3위)** 대조 구도.
- **Transformer 특이점**: 1위는 절대 위치 임베딩 대신 **ALiBi 상대위치 bias**로 가변 시퀀스 길이/외삽에 강건 (확인: CNN_GRU_base.py:104-187).
- **헤드 패러다임 차이**: 베이스라인/1위 = **직접 좌표 회귀(MSE)**, TDTracker = **이산화 분류(SimDR/KL-div) + heatmap argmax** → 후자는 confidence 산출로 후처리(눈감음/저신뢰 대체) 가능 (확인: metrics.py:104-166, post_process.py).

## 7. 강점·한계

### 강점
- **베이스라인**: 깔끔한 Tonic 기반 파이프라인 + 디스크 캐싱 + MLflow, 재현·확장 용이. 모델 단순(<수십만 파라미터 추정)으로 경량.
- **1위**: ALiBi Transformer로 장기 시간의존성·가변길이 강건. 강한 증강(Jitter: shift/flip/cutout), warm-up 후 inference time 계측 → 정확도·지연 동시 의식.
- **TDTracker**: (a) spatial/temporal 분리 3D-CNN으로 시공간 분해, (b) Mamba 선형복잡도 SSM + RNN형 step 추론 → 스트리밍/온디바이스 친화, (c) SimDR 분류로 confidence 기반 후처리(눈감음·저신뢰 보정)까지 일관 설계.

### 한계
- **베이스라인**: 단일 GRU·MaxPool로 표현력 제한, 좌표 회귀라 outlier/눈감음 처리 없음. GRU input 36192 하드코딩(해상도 변경 시 깨짐, 확인: BaselineEyeTrackingModel.py:15).
- **1위**: CNN 채널 512·Transformer 6층 → 파라미터/연산 큼(경량화/양자화 부담, 추정). 입력채널 2 하드코딩(라인 16). 미사용 코드(Mamba/EfficientNet/mixup) 잔존으로 가독성 저하. tsf=0.2 5-fold 분기 등 추론 로직 복잡(확인: test.py:25-107).
- **TDTracker**: `selective_scan_seq` 파이썬 for-loop(라인 283-287) → GPU 학습 비효율(추론은 step 캐시로 완화). 오프라인 .h5 전체를 RAM 로드(train.py:158-166) → 대용량 시 메모리 한계. dataprocess 경로 하드코딩(라인 264-272). FFT layer 등 사장 코드 다수.
- **공통**: ground truth 100Hz, 80x60 저해상도 좌표 → 미세 동공 변위 한계. close 라벨 활용은 TDTracker만 적극적.

## 8. 우리 프로젝트(PRJXR-HBTXR) 시사점 — "XR 시선추적 + FPGA 저지연 on-device 가속" (목표 추정)

> 우리 프로젝트 목표를 "XR용 이벤트 기반 시선추적 모델의 경량화·양자화·FPGA 이식"으로 추정. 아래는 본 코드베이스에서 도출한 직접 활용점.

1. **백본 선택 — FPGA 친화도 순위(추정)**:
   - **TDTracker의 Mamba/SSM이 FPGA 이식 1순위 후보**. 이유: (a) selective scan이 순수 PyTorch 행렬연산(MAC+exp+softplus)으로 분해되어 HLS 매핑 명확(확인: TDTracker.py:240-293), (b) `step()`/`ssm_step()`가 **상수시간 RNN형 재귀**(h, conv cache만 유지)라 스트리밍 저지연 파이프라인에 적합(확인: 라인 317-381). Transformer(1위)의 softmax attention 대비 메모리 대역폭·면적 유리(추정).
   - 단, Mamba의 `exp`/`softplus`/element-wise gating(silu)은 FPGA에서 **LUT 근사·고정소수점화** 필요(우리 on-device 스킬의 LayerNorm/Softmax/GELU 데이터패스 설계와 직결).
2. **입력 표현 — binary(이진 누적) 표현 채택 권장**: 상위 2솔루션 모두 binary map 수렴(확인). voxel 대비 **정수/비트 단위 표현**으로 FPGA 입력 버퍼·곱셈 제거 가능 → 입력단 양자화 자연스러움. 80x60x2 저해상도라 on-chip BRAM 상주 가능(추정: 80*60*2 = 9600 elem).
3. **헤드 — SimDR 분류 헤드의 하드웨어 이점**: 좌표 회귀(MSE) 대신 80/60 bin **argmax 분류**(확인: TDTracker)는 FPGA에서 비교기 트리(argmax)만으로 디코딩 가능하고 confidence를 부산물로 얻어 **눈감음 게이팅** 구현이 저렴. 회귀 FC 대비 양자화 강건성도 유리(추정).
4. **양자화 전략**: 1위 CNN 채널 512는 과대 → **TDTracker급(32/64/128 채널) + 3D depthwise/separable 분리 구조**가 PTQ/QAT INT8 친화(추정). spatial(1×k×k)/temporal(3×3×3) 분리는 곱셈 재사용·systolic array 매핑에 유리.
5. **저지연 평가 관행 차용**: 1위 test.py의 warm-up 후 inference time 측정 + `inference_time.txt`(확인: test.py:144-194), TDTracker의 step 캐시 추론 → **우리 FPGA 레이턴시 벤치마크/사이클 리포트와 직접 대응**. Neurobench 메모리 평가(확인: 베이스라인 README:180)는 우리 BRAM/면적 목표 설정에 참고 지표.
6. **후처리 경량화**: TDTracker post_process(confidence<0.5·이벤트수<0.09 대체, binary_dilation)는 **호스트(CPU/ARM) 측 경량 후처리**로 분리 가능 → FPGA는 추론만, 후처리는 SW. 단 binary_dilation/scipy는 FPGA 미적합, 단순 임계+직전값 hold 로직으로 RTL화 가능(추정).
7. **재현 파이프라인**: Tonic SlicedDataset/DiskCached 캐싱(확인)은 학습 단계용이며 추론(FPGA)과 무관 → 이식 시 전처리(slice+binary map)만 C/HLS로 포팅. dataprocess/3etplus.py가 전처리 명세(SLIDING=10000us, ToFrame+ToBinaRep n_bits=4)를 라인 단위로 제공(확인) → RTL 입력단 사양 직접 도출 가능.
