## Runtime Policy

- `FACET_DEVICES=0` is used for EPNet/FACET baseline.
- `FACET_DEVICES=1` is used for HBTXR.
- `FACET_DISABLE_CUDNN=1` is retained for the RTX 5080/cuDNN runtime issue observed earlier.
- Both scripts wait for the same manifest file, run a `DavisEyeEllipseDataset` train/val smoke check, and then start independently.
- The current full expansion process continues on GPU0 until the manifest is produced. The EPNet script only starts after that gate, so it should not conflict with expansion.

## Goal Addition

The reproduction goal is no longer only "reproduce FACET EPNet metrics." It now includes:

1. Reproduce FACET EPNet baseline on the full U-Net-expanded dataset.
2. Train HBTXR-DeiT on the same full dataset in parallel on GPU1.
3. Compare EPNet/FACET and HBTXR using the same validation split and metric scripts.
