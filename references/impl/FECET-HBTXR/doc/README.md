# Documentation Policy

This directory stores lightweight project history for `FECET-HBTXR`.

## Files

- `UPDATE_HISTORY.md`: dated implementation history and major repository changes
- `CONVERSATION_HISTORY.md`: concise decision log distilled from user-assistant interactions
- `PROGRESS_CHECKLIST.md`: plan-vs-progress status and current validation state
- `FACET_QUANT_COMPARISON.md`: end-to-end FACET vs FECET-HBTXR comparison workflow

## Update Rule

- Append new dated entries instead of rewriting older decisions.
- Keep the conversation log concise. Record requests, constraints, and accepted decisions rather than full transcripts.
- Keep the progress checklist explicit about what is implemented, what is only test-validated, and what is still pending on the real dataset.
- Do not store raw dataset contents, generated manifests, or temporary experiment outputs in this directory.
